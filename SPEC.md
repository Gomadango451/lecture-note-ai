# SPEC.md — 講義音声ノートAI（DAY5）

## 1. 概要

大学の講義や自習の録音音声を入力すると、以下の3つを自動生成するWebアプリ。

- 文字起こし（全文）
- 要約（3行程度）
- 要点（箇条書き 3〜5個）

## 2. 技術スタック

| 項目 | 内容 |
|------|------|
| フロントエンド | HTML / CSS / JavaScript（単一ファイル構成） |
| バックエンド | なし（フロントエンドのみ） |
| AI API | Gemini API（Google） |
| 公開方法 | GitHub Pages |
| APIキー管理 | ユーザーが初回入力 → localStorage に保存（リポジトリには含めない） |

## 3. 画面構成・ユーザー導線

```
[1] APIキー入力欄（初回のみ。保存済みなら非表示 or 折りたたみ）
[2] 講義名入力欄（任意）
[3] 音声ファイル選択ボタン（mp3 / m4a / wav / ogg 対応）
[4] 実行ボタン「文字起こし＆要約する」
[5] ローディング表示（処理中）
[6] 文字起こし結果エリア＋コピーボタン
[7] 要約結果エリア＋コピーボタン
[8] 要点（箇条書き）エリア＋コピーボタン
```

## 4. 入力仕様

| 項目 | 内容 |
|------|------|
| 講義名 | テキスト入力（任意） |
| 音声ファイル | ファイル選択（推奨：30秒〜2分の短いファイルでテスト） |
| 対応形式 | mp3, m4a, wav, ogg, flac, aac |
| サイズ制限 | Gemini inline: 20MB以下を推奨（それ以上はFiles API） |

## 5. 出力仕様

| 項目 | 内容 |
|------|------|
| 文字起こし | 音声の全文テキスト |
| 要約 | 3行程度の簡潔な要約 |
| 要点 | 箇条書き 3〜5個 |
| コピー機能 | 各出力エリアにコピーボタン設置 |

## 6. Gemini API仕様

- **使用モデル：** `gemini-2.0-flash`（※公式モデル一覧を都度確認すること）
- **公式モデル一覧：** https://ai.google.dev/gemini-api/docs/models?hl=ja
- **音声処理ドキュメント：** https://ai.google.dev/gemini-api/docs/audio
- **エンドポイント：** `https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent`
- **認証：** APIキーをクエリパラメータ `?key=YOUR_API_KEY` で渡す
- **音声データの渡し方：** Base64エンコードしてリクエストに含める（inline_data）

## 7. APIキーの取り扱い

- GitHubリポジトリには絶対に含めない
- ユーザーが画面上で入力 → `localStorage.setItem('gemini_api_key', key)` で保存
- 次回アクセス時は `localStorage.getItem('gemini_api_key')` で読み込み

## 8. エラー処理

| ケース | 表示内容 |
|--------|---------|
| APIキー未入力 | 「APIキーを入力してください」 |
| ファイル未選択 | 「音声ファイルを選択してください」 |
| API通信失敗（401） | 「APIキーが無効です。確認してください」 |
| API通信失敗（400） | 「リクエストエラー。ファイル形式やサイズを確認してください」 |
| その他エラー | 「エラーが発生しました：{エラー内容}」 |

## 9. UI要件

- スマホで操作できるレスポンシブデザイン
- シンプルで直感的なレイアウト
- ローディング中はボタンを非活性にする

## 10. 非機能要件

- 個人情報・機密情報を含む音声はアップロードしない（短いサンプルで検証）
- 提出物（スクショ等）に個人情報を写さない
- 公開リポジトリにAPIキーを含めない（必須）
